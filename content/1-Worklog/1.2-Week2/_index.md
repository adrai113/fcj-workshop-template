### Objectives of Week 2:

- Follow the "First Cloud AI Journey" roadmap, focusing on core AWS services.
- Gain deep understanding of Amazon EC2 compute service, Amazon S3 object storage, and IAM identity management.
- Ability to independently deploy a secure virtual server and assign access rights adhering to the "Least Privilege" principle.

---

### Tasks Implemented in the Week:

- **Identity Management (IAM):** 
  - Researched IAM authorization mechanisms. Practiced creating IAM Users, User Groups, and configuring passwords/Access Keys. 
  - Analyzed IAM Policy JSON structures and assigned Policies to Groups. 
  - Learned and practiced IAM Role delegation instead of using static Access Keys.
- **Cloud Computing (Amazon EC2):** 
  - Initialized EC2 virtual servers, explored Amazon Machine Images (AMIs), and Instance Types (Compute-optimized, Memory-optimized, etc.). 
  - Configured Key Pairs (SSH Keys) for secure server access. 
  - Set up basic Security Groups (Opened port 22 for SSH and port 80 for HTTP).
- **Cloud Storage (Amazon S3):** 
  - Created S3 Buckets, practiced uploading and managing Objects. 
  - Learned the basics of Block Storage (EBS) attached to EC2 vs Object Storage (S3).
- **Notes & Training:** Continued following the video roadmap, systematizing knowledge on how these services interact.

---

### Achievements:

- Mastered the process of operating and initializing a complete EC2 virtual server from scratch.
- Clearly understood the importance of IAM in system security and how to apply multi-layer security principles.
- Initially got acquainted with S3 storage, distinguishing the use cases between S3 and EBS.
